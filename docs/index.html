Critical Evaluation

Advantages/disadvantages of the REST compaired to SOAP and XML RPC?

REST represents the representational state transfer. "An architectural style for networked hypermedia applications, it is primarily used to build Web services that are lightweight, maintainable, and scalable"(Vaqqas, 2014).SOAP is the acronym of simpel object access protocol. "SOAP is a standards-based Web services access protocol that has been around for a while and enjoys all of the benefits of long-term use"(Mueller, 2013).RPC means Remote procedure call. "Remote Procedure Call (RPC) is a protocol that one program can use to request a service from a program located in another computer on a network without having to understand the network's details"(Rouse, 2016).

REST vs SOAP

SOAP is not simple as its name. SOAP is based on standards. Developers need to understand SOAP works. Also SOAP depends on programming languages. It is a heavy weight protocol. On the other hand REST is simpler than SOAP. Need no basic knowledge. A RESTful service can be implemented easily without any standards. It is light weight and greately extensible and scalable.

REST vs RPC

RPC is a protocol that clients can use to request a function or a procedure in a program. A program may be in the same or another netowrk. REST architecture is similar to RPC. REST is the architecture style on the other hand, RPC is a protocol. It is hard to compare since they are of different type. REST can be implemented in top of the RPC system. Just to create GET,POST,PUT and DELETE procedures.

Extending and Improving the Application

This system is a web api. The system may be extended in various way. We can build a website showing live currency conversion based on this API. Also we can even build mobile applications using this api. So that people can check on the go using their mobile devices.

Promotion and code re-use

I have written the rquired functions in the function.php file for further reusable purpose. isInISO() ,getElementbyID() and getDOMElementbyID() are the useful functions. isInISO() function is called when user wants to insert new currency. It checks if the input currency is in ISO supported currencies. getElementbyID() is used to get a simple xml element which is defined by id attribute. getDOMElementbyID() is used to get a dom xml element which is defiend by id attribute. These functions are useful and flexible that can be use for any xml document. The user only has to set id attribute and can get the node or element by id value.
